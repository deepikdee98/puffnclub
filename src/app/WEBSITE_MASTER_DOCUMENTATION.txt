1. WEBSITE OVERVIEW
================================================================================

The Puffn E-Commerce Website is a modern, customer-facing shopping platform
built with Next.js 14 and React. It provides a complete shopping experience
including product browsing, cart management, user authentication, order
tracking, and wishlist functionality.

KEY CHARACTERISTICS:
- Fully responsive design (mobile, tablet, desktop)
- Server-side rendering with Next.js 14
- React Bootstrap UI components
- SCSS styling for custom designs
- Real-time cart and wishlist management
- OTP-based authentication (Phone & Email)
- Integrated payment and checkout system
- Product reviews and ratings
- Coupon/discount system
- Category-based product filtering

================================================================================
2. ARCHITECTURE & TECHNOLOGY STACK
================================================================================

FRONTEND FRAMEWORK:
- Next.js 14 (App Router)
- React 18
- TypeScript

UI FRAMEWORK:
- React Bootstrap
- SCSS Modules
- React Icons (Feather Icons)

STATE MANAGEMENT:
- React Context API (Auth, Cart, Wishlist)
- Local component state with useState
- Custom hooks for data fetching

API INTEGRATION:
- RESTful API calls using fetch
- Centralized API endpoints configuration
- JWT token-based authentication

ANIMATIONS:
- Custom scroll-based animations (FadeUp, FlipX, FlipY, ZoomIn)
- React Bootstrap transitions

UTILITIES:
- React Toastify for notifications
- Axios for some API calls
- Custom utility functions for data normalization

6. INTEGRATION POINTS
================================================================================

BACKEND API:
- Base URL: http://localhost:5000/api
- All endpoints defined in: services/api.ts
- Uses JWT Bearer token authentication
- RESTful API architecture

KEY API ENDPOINTS:
- /website/auth/*          - Authentication
- /website/products/*      - Product data
- /website/cart/*          - Cart operations
- /website/wishlist/*      - Wishlist operations
- /website/orders/*        - Order management
- /website/addresses/*     - Address management
- /coupons/*               - Coupon validation

DATA FLOW:
1. User actions trigger service calls
2. Services make API requests with auth tokens
3. Context providers manage global state
4. Components re-render with updated data
5. Toast notifications inform users
